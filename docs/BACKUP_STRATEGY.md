# Backup Strategy

This document outlines the backup and recovery procedures for the SequoIA production deployment.

---

## Overview

| Component | Data Type | Retention | Backup Frequency | Recovery Time |
|-----------|-----------|-----------|------------------|---------------|
| PostgreSQL | User/org data, config | Forever | Daily | < 1 hour |
| ClickHouse | Time-series metrics | 48h/90d/forever tiered | Weekly | < 4 hours |
| Kafka | Message logs | 24 hours | Not backed up | Self-healing |
| Configuration | .env, fibers.yaml | Forever | On change | < 15 min |

---

## 1. PostgreSQL Backup

PostgreSQL stores user accounts, organizations, fiber assignments, and Django config.

### Daily Backup Script

Create `/opt/sequoia/scripts/backup-postgres.sh`:

```bash
#!/bin/bash
set -e

BACKUP_DIR="/mnt/backups/postgres"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/sequoia_${TIMESTAMP}.sql.gz"
RETENTION_DAYS=30

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Dump database
docker exec postgres pg_dump -U sequoia -d sequoia | gzip > "$BACKUP_FILE"

# Verify backup
if [ -s "$BACKUP_FILE" ]; then
    echo "Backup created: $BACKUP_FILE ($(du -h "$BACKUP_FILE" | cut -f1))"
else
    echo "ERROR: Backup file is empty!"
    exit 1
fi

# Remove old backups
find "$BACKUP_DIR" -name "sequoia_*.sql.gz" -mtime +${RETENTION_DAYS} -delete
echo "Cleaned backups older than ${RETENTION_DAYS} days"
```

### Schedule with Cron

```bash
# Run daily at 2 AM
0 2 * * * /opt/sequoia/scripts/backup-postgres.sh >> /var/log/sequoia-backup.log 2>&1
```

### Restore Procedure

```bash
# Stop the backend to prevent writes
docker stop platform-backend

# Restore from backup
gunzip -c /mnt/backups/postgres/sequoia_20260219_020000.sql.gz | \
  docker exec -i postgres psql -U sequoia -d sequoia

# Restart backend
docker start platform-backend
```

## 2. ClickHouse Backup

ClickHouse stores speed/count time-series data with automatic TTL tiering:
- `speed_hires` / `count_hires`: 48 hours (high resolution)
- `speed_1m` / `count_1m`: 90 days (1-minute aggregates)
- `speed_1h` / `count_1h`: Forever (1-hour aggregates)

### Weekly Backup Script

Create `/opt/sequoia/scripts/backup-clickhouse.sh`:

```bash
#!/bin/bash
set -e

BACKUP_DIR="/mnt/backups/clickhouse"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
TABLES="speed_1h count_1h speed_1m count_1m fiber_cables fiber_incidents"
RETENTION_WEEKS=8

mkdir -p "$BACKUP_DIR"

for TABLE in $TABLES; do
    BACKUP_FILE="${BACKUP_DIR}/${TABLE}_${TIMESTAMP}.native.gz"

    docker exec clickhouse clickhouse-client \
        --query="SELECT * FROM sequoia.${TABLE} FORMAT Native" | \
        gzip > "$BACKUP_FILE"

    echo "Backed up $TABLE: $(du -h "$BACKUP_FILE" | cut -f1)"
done

# Remove old backups (keep 8 weeks)
find "$BACKUP_DIR" -name "*.native.gz" -mtime +$((RETENTION_WEEKS * 7)) -delete
```

### Restore Procedure

```bash
# Restore a single table
gunzip -c /mnt/backups/clickhouse/speed_1h_20260219.native.gz | \
  docker exec -i clickhouse clickhouse-client \
    --query="INSERT INTO sequoia.speed_1h FORMAT Native"
```

### Alternative: ClickHouse Native Backup

For large deployments, use ClickHouse's native backup:

```sql
-- Create backup
BACKUP TABLE sequoia.speed_1h, sequoia.count_1h
TO Disk('backups', 'weekly_20260219');

-- Restore
RESTORE TABLE sequoia.speed_1h, sequoia.count_1h
FROM Disk('backups', 'weekly_20260219');
```

## 3. Configuration Backup

### What to Back Up

```
/opt/sequoia/
├── .env                     # Production secrets
├── config/
│   └── fibers.yaml          # Fiber configurations
├── infrastructure/
│   └── clickhouse/
│       └── cables/*.json    # Fiber cable geometries
└── data/
    └── calibration/         # AI model calibration files
```

### Backup Script

```bash
#!/bin/bash
BACKUP_DIR="/mnt/backups/config"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"

tar -czf "${BACKUP_DIR}/config_${TIMESTAMP}.tar.gz" \
    /opt/sequoia/.env \
    /opt/sequoia/config/ \
    /opt/sequoia/infrastructure/clickhouse/cables/ \
    /opt/sequoia/data/calibration/

echo "Config backup: ${BACKUP_DIR}/config_${TIMESTAMP}.tar.gz"
```

## 4. Kafka (No Backup Required)

Kafka is a streaming system with 24-hour retention. Data flows through to ClickHouse automatically.

**If Kafka data is lost:**
- Recent data (< 24h): Lost, will be regenerated by DAS hardware
- Historical data: Safe in ClickHouse
- System recovers automatically when Kafka restarts

## 5. Disaster Recovery

### Full System Restore Procedure

1. **Provision new server** with Docker installed

2. **Restore configuration:**
   ```bash
   tar -xzf config_backup.tar.gz -C /opt/sequoia/
   ```

3. **Create data directories:**
   ```bash
   mkdir -p /mnt/{kafka-data,clickhouse-data,postgres-data,redis-data}
   chown -R 1001:1001 /mnt/kafka-data
   chown -R 101:101 /mnt/clickhouse-data
   chown -R 999:999 /mnt/postgres-data
   ```

4. **Start infrastructure services:**
   ```bash
   docker compose up -d kafka postgres clickhouse redis
   ```

5. **Restore PostgreSQL:**
   ```bash
   gunzip -c postgres_backup.sql.gz | docker exec -i postgres psql -U sequoia -d sequoia
   ```

6. **Restore ClickHouse tables:**
   ```bash
   for file in /mnt/backups/clickhouse/*.native.gz; do
       table=$(basename "$file" | cut -d'_' -f1-2)
       gunzip -c "$file" | docker exec -i clickhouse clickhouse-client \
           --query="INSERT INTO sequoia.${table} FORMAT Native"
   done
   ```

7. **Start remaining services:**
   ```bash
   docker compose up -d
   ```

8. **Verify:**
   ```bash
   curl http://localhost:8001/api/health
   ```

## 6. Backup Verification

### Weekly Verification Checklist

- [ ] PostgreSQL backup file exists and is > 1KB
- [ ] ClickHouse backup files exist for all tables
- [ ] Test restore to staging environment monthly
- [ ] Verify backup retention policy is working (old files deleted)

### Automated Verification Script

```bash
#!/bin/bash
# Run weekly to verify backup integrity

POSTGRES_LATEST=$(ls -t /mnt/backups/postgres/*.sql.gz | head -1)
CH_LATEST=$(ls -t /mnt/backups/clickhouse/speed_1h_*.native.gz | head -1)

# Check file sizes
if [ $(stat -f%z "$POSTGRES_LATEST") -lt 1000 ]; then
    echo "WARNING: PostgreSQL backup seems too small"
    exit 1
fi

if [ $(stat -f%z "$CH_LATEST") -lt 1000 ]; then
    echo "WARNING: ClickHouse backup seems too small"
    exit 1
fi

echo "Backup verification passed"
```

## 7. Offsite Backup (Recommended)

For production, sync backups to offsite storage:

```bash
# Sync to S3-compatible storage
aws s3 sync /mnt/backups/ s3://sequoia-backups/ --delete

# Or rsync to another server
rsync -avz /mnt/backups/ backup-server:/mnt/sequoia-backups/
```
